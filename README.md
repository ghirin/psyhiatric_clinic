# 🏥 Система учёта пациентов психиатрической клиники

**Статус:** Учебный пет-проект (активная разработка)  
**Роль:** Fullstack разработчик (единственный автор)  
**Стек:** Python 3, Django, PostgreSQL, HTML, CSS, JavaScript, Docker  
**Исходный код:** [github.com/ghirin/psyhiatric_clinic](https://github.com/ghirin/psyhiatric_clinic)

### 🎯 Задача проекта
Разработать современную веб-систему для замены устаревшего desktop-приложения (Delphi 7), которое использовалось для учёта пациентов в психиатрической клинике. Основная цель — обеспечить безопасное хранение медицинских данных с разграничением доступа.

### 🔧 Мои задачи и решения
1. **Миграция с Delphi 7** → Спроектировал реляционную схему БД (PostgreSQL) и реализовал бизнес-логику на Django, полностью переосмыслив архитектуру.
2. **Безопасность медицинских данных** → Внедрил:
   - Шифрование чувствительных полей в БД.
   - Логирование критических действий (доступ к картам, изменения).
   - Разграничение ролей через пользовательскую модель (`users`).
3. **Пользовательский интерфейс** → Разработал удобные формы ввода, таблицы с быстрым поиском и горячими клавишами (вынесенными во внешний JS).
4. **Деплой и автоматизация** → Написал Dockerfile, docker-compose.yml и набор bash-скриптов (`apply_migrations.sh`, `check_project.sh`) для быстрого развёртывания.

### 💡 Результат и ценность для бизнеса
- ✅ Код полностью открыт, система готова к запуску в изолированной сети клиники.
- ✅ Повышена безопасность и надёжность учёта по сравнению с Delphi-решением.
- ✅ Упрощено внесение изменений и масштабирование (благодаря Django-архитектуре).

- Чистая структура проекта (core, patients, users), готовая к расширению.
